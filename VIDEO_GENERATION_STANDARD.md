# 视频生成标准 - 8秒分镜规范

## 标准规格

### 基础参数
- **标准时长**: 8 秒/分镜
- **分辨率**: 1920×1080 (1080p)
- **帧率**: 24 fps
- **编码**: H.264
- **码率**: 8-12 Mbps

### Seedance 2.0 约束
- 图像参考: ≤9 张
- 视频参考: ≤3 个
- 音频参考: ≤3 个
- 提示词长度: ≤5000 字符
- **时长范围**: 4-15 秒（推荐 8 秒）

---

## 8秒分镜设计原则

### 1. 单一动作分镜（8秒）

**适用场景**: 简单动作、单一镜头
**时间分配**:
```
0-2s: 建立镜头（Establishing）
2-6s: 主要动作（Main Action）
6-8s: 收尾/过渡（Transition）
```

**示例**:
```
0-2s: 角色从远处走来，镜头推进
2-6s: 角色走到中央，停下站定
6-8s: 角色转身面向镜头
```

### 2. 复合动作分镜（8秒）

**适用场景**: 连续动作、情节推进
**时间分配**:
```
0-2s: 起（Setup）
2-5s: 承（Development）
5-8s:转（Turn）
```

**示例**:
```
0-2s: 舞者准备姿势
2-5s: 舞者旋转起舞
5-8s: 舞者跃起腾空
```

### 3. 高潮分镜（8秒）

**适用场景**: 关键时刻、情感爆发
**时间分配**:
```
0-2s: 蓄力（Build-up）
2-4s: 爆发（Climax）- 可用慢动作
4-8s: 余韵（Aftermath）
```

**示例**:
```
0-2s: 舞者加速旋转
2-4s: 舞者跃起（慢动作 0.5x）
4-8s: 舞者落地，裙摆飘落
```

## 分镜规划策略

### 短剧本（30-60秒）
- **分镜数**: 4-6 个
- **每镜**: 8 秒
- **总时长**: 32-48 秒
- **策略**: 精简情节，突出重点

### 中等剧本（1-2分钟）
- **分镜数**: 8-12 个
- **每镜**: 8 秒
- **总时长**: 64-96 秒
- **策略**: 完整叙事，节奏流畅

### 长剧本（2-5分钟）
- **分镜数**: 15-30 个
- **每镜**: 8 秒
- **总时长**: 120-240 秒
- **策略**: 分章节，多场景切换

## 性能优化建议

### 1. 批量生成
```python
# 7个8秒分镜并行生成
tasks = []
for frame in frames:
    task = generate_video(
        frame_id=frame.id,
        duration=8,  # 固定8秒
        provider="seedance"
    )
    tasks.append(task)

# 并行执行，总耗时 = 最慢的一个（约120秒）
results = await asyncio.gather(*tasks)
```

### 2. 时长控制
```python
# 提示词中明确指定8秒
prompt = f"""
{timeline_8s}

时长: 8秒
帧率: 24fps
总帧数: 192帧

```

### 3. 质量保证
```python
# 验证生成的视频时长
def validate_video_duration(video_path):
    cap = cv2.VideoCapture(video_path)
    fps = cap.get(cv2.CAP_PROP_FPS)
    frame_count = cap.get(cv2.CAP_PROP_FRAME_COUNT)
    duration = frame_count / fps
    cap.release()
    
    # 允许 ±0.5秒误差
    assert 7.5 <= duration <= 8.5, f"时长异常: {duration}秒"
```

## 最佳实践

### ✅ 推荐做法
1. 所有分镜统一 8 秒时长
2. 时间轴提示词精确到秒
3. 关键动作放在 2-6 秒区间
4. 慢动作控制在 2 秒以内
5. 前后分镜留 0.5 秒过渡空间

### ❌ 避免做法
1. 不要在 8 秒内塞入过多动作
2. 不要频繁切换场景（≤3 个场景/镜）
3. 不要使用过长的慢动作（>3 秒）
4. 不要忽略时间轴分段
5. 不要超过 Seedance 约束限制

## 工具支持

### 自动分镜规划器
```python
class StoryboardPlanner:
    def plan_8s_frames(self, script: str, target_duration: int = 60):
        """
        根据剧本自动规划8秒分镜
        
        Args:
            script: 原始剧本
            target_duration: 目标总时长（秒）
        
        Returns:
            分镜列表，每个8秒
        """
        # 计算分镜数
        num_frames = target_duration // 8
        
        # LLM 分析剧本，提取关键情节点
        plot_nodes = self.llm.extract_plot_nodes(script)
        
        # 将情节点分配到分镜
        frames = self.distribute_nodes_to_frames(
            plot_nodes,
            num_frames,
            duration_per_frame=8
        )
        
        return frames
```

## 总结

采用 **8 秒标准分镜** 的优势：

1. ✅ **统一规范**: 所有分镜时长一致，便于管理
2. ✅ **成本可控**: 固定时长，成本可预测
3. ✅ **质量稳定**: 8 秒是视频生成模型的最佳时长
4. ✅ **剪辑友好**: 统一时长便于后期剪辑和调整
5. ✅ **性能优化**: 批量并行生成，效率最高

推荐所有项目采用 8 秒标准分镜规范。
